## Stage 3 — Implementation

```r
library(tidyverse)
library(fixest)
library(modelsummary)

# ── 0. Inspect balance ──────────────────────────────────────────────────────
df |>
  group_by(treated) |>
  summarize(
    n_stores      = n_distinct(store),
    n_months      = n_distinct(month),
    mean_sales_pre  = mean(sales[post == 0]),
    mean_sales_post = mean(sales[post == 1])
  )

# ── 1. Main DiD estimate ────────────────────────────────────────────────────
# Note: `treated` and `post` are each absorbed by their respective FE.
# Only the interaction term varies across both dimensions — that's our ATT.
model <- feols(
  sales ~ treated:post | store + month,
  data    = df,
  cluster = ~store   # cluster at the unit (store) level
)

summary(model)

# ── 2. Effect size + 95% CI ─────────────────────────────────────────────────
coefplot(model, main = "DiD Estimate: Effect of Inventory System on Sales")

att_est <- coef(model)["treated:post"]
att_ci  <- confint(model)["treated:post", ]

cat(sprintf(
  "\nATT: %.3f  (95%% CI: %.3f, %.3f)\n",
  att_est, att_ci[1], att_ci[2]
))

# ── 3. Results table ────────────────────────────────────────────────────────
modelsummary(
  model,
  stars    = TRUE,
  gof_map  = c("nobs", "r.squared"),
  title    = "Effect of New Inventory System on Sales (DiD)"
)

# ── 4. Pre-trends / event study ─────────────────────────────────────────────
# Create time-to-treatment variable (month 12 = 0, reference period = -1)
df <- df |>
  mutate(time_to_treat = month - 12)

es_model <- feols(
  sales ~ i(time_to_treat, treated, ref = -1) | store + month,
  data    = df,
  cluster = ~store
)

iplot(
  es_model,
  main  = "Event Study: Effect of Inventory System on Sales",
  xlab  = "Months Relative to Treatment (Month 12)",
  ylab  = "Estimated Effect on Sales",
  ref.line = TRUE  # draws vertical line at t = -1 reference
)
# What to look for:
#   PRE-PERIOD  (t < 0): coefficients should hover near zero — supports parallel trends
#   POST-PERIOD (t ≥ 0): size and shape of the treatment effect
```

**What each piece does:**

- `treated:post` — the DiD interaction. `feols` with `| store + month` absorbs all time-invariant store differences and all common time shocks, leaving only this cross-variation.
- `cluster = ~store` — standard errors account for serial correlation within stores over time. This is almost always right for panel DiD.
- The event study (`i()` syntax) is your most important diagnostic — it shows you both the pre-trend test and the dynamic effect trajectory in one plot.
